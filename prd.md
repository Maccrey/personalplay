문항 8~12개 중심 (짧고 빠른 몰입형)

```markdown
PersonaPlay (퍼소나플레이) — 비즈니스 중심 PRD (AdSense 수익화 최적화)

## 1. 한줄 요약

재미 중심의 모바일 우선 성향 테스트 플랫폼으로, SNS 공유와 재참여를 유도해 유의미한 페이지뷰를 만들어 Google AdSense 및 브랜드 제휴로 수익을 창출한다.

## 2. 비즈니스 목표 (Business Objectives)

- 단기(런칭 3개월): 일일 UV 10,000 / 페이지뷰 20,000 달성, AdSense로 월수익(예상) 500~1,500 USD 수준 확보
- 중기(6~12개월): 충성 유저 기반 구축으로 재방문율 25% 이상, EPMV/UV 개선을 통해 월간 광고 수익 3,000~10,000 USD 목표
- 장기(1년+): 브랜드 제휴 및 프리미엄 리포트를 통한 다각화 수익 모델 정착, 광고 의존도 축소 및 ARPU 상승

## 3. 핵심 가치 제안

- 즉시 공유 가능한 결과 카드(시각화된 이미지 + 한줄 카피)
- 모바일 최적화된 짧고 중독성 있는 테스트(문항 8~12개)
- 반복 방문을 유도하는 컬렉션·랭킹·친구 비교 기능

## 4. 수익화 전략 (AdSense 중심)

목표: 빠른 트래픽 확보 후 광고 수익화(AdSense)로 기본 수익 선점, 이후 고수익 채널(브랜드 협업, 프리미엄 리포트) 확장

- AdSense 구현 방식

  - 반응형 광고(Responsive) + 고정형 스티키(앵커) + 네이티브 인-article 조합
  - 광고 공급망 확장: 초기엔 AdSense 직접 연동, 이후 Google Ad Manager(헤더비딩 포함) 연동으로 수익 극대화
  - 유저 세그먼트에 따른 맞춤형 광고(콘텐츠 태깅 기반) 적용

- 권장 광고 슬롯(모바일 우선)

  1.  결과 페이지 상단(프리로더 직후, 콘텐츠보다 눈에 띄지 않는 leaderboard) — 1 슬롯 (반응형)
  2.  결과 카드 하단(공유 CTA 위) — 인-article / 네이티브 슬롯
  3.  결과 페이지 하단(스크롤 끝) — 배너 또는 자동로딩 광고
  4.  스티키 앵커(하단 고정, 320x50 또는 반응형) — 빈도 제한으로 UX 보장
  5.  선택적 인터스티셜(페이지 전환 시, 전략적으로 제한된 상황에서만) — 사용 시 AdSense 정책과 앱스토어/웹 가이드라인 준수 필수

- 광고 정책 & UX 원칙
  - AdSense 정책 준수: 유도 클릭 금지, 오해의 소지가 있는 배치 금지, 성인/민감한 카테고리 광고 회피
  - 클릭 유도성 문구 금지(예: "광고 클릭하면~"), 광고와 콘텐츠 구분 명확화
  - 광고 밀도: 화면 당 광고 노출 최소화, 모바일 화면에서 콘텐츠 대비 광고 비율(예: 1:3 이상) 유지
  - 페이지 속도 우선: 광고 lazy-load, viewability 최적화

## 5. 제품/기능 요구사항 (Business & Product)

### 5.1 필수 기능(런칭 MVP)

- 빠른 테스트 실행: 문항 8~12개, 평균 소요 60초 이하
- 결과 카드 자동 생성(OG 태그 포함) — SNS에서 미리보기 최적화
- 공유 버튼: 인스타 스토리, 틱톡, X, 네이버 블로그 등
- AdSense 광고 슬롯(위 4개) 기본 구성 및 A/B 테스트 구조화
- 기본 분석: Google Analytics, 광고 노출/클릭 추적, 페이지별 RPM 집계

### 5.2 운영 기능

- CMS: 테스트/문항/결과 콘텐츠 편집 인터페이스
- 캠페인 관리: 브랜드 제휴 테스트 스펙 등록 및 승인 워크플로우
- 유저 관리: 익명 세션 + 로그인 선택(소셜 로그인) 지원

### 5.3 규정·정책·법적 요구사항

- 개인정보·동의: 쿠키 동의 배너(CMP) 및 GDPR/CCPA 대응
- AdSense 정책 검토 체크리스트(광고 위치, 콘텐츠 카테고리, 클릭 유도 금지)

## 6. UX / UI 가이드라인 (광고를 고려한 설계)

- 결과 중심 레이아웃: 결과 이미지 → 핵심문구 → 공유 CTA(광고보다 우선) → 관련 콘텐츠/광고
- 공유 UX: 공유 전에 광고가 먼저 노출되지 않도록 설계
- 광고 시인성: 광고와 콘텐츠는 시각적으로 구분, 광고 태그에 aria-label 포함
- 광고 위치 실험: A/B 테스트로 CTR/VIEWABILITY/EPMV 비교

## 7. 기술 스택 및 구현 고려사항

- Front: Next.js (SSR/ISR 활용해 SEO + OG 최적화), TailwindCSS
- Back: Node.js + Express (API), 또는 Firebase Functions로 빠른 MVP 가능
- DB: Supabase/Firebase (유저 세션, 게시물 수집)
- Ads: Google AdSense(초기), Google Ad Manager + Header Bidding(확장)
- AI: OpenAI API(결과 문장/캐릭터 카피 생성), 이미지 생성은 외부 이미지 API 또는 자체 템플릿 사용
- Analytics: GA4 + 서버로그(광고 클릭/노출 매핑), BigQuery(장기 분석)
- Consent: TCF v2.0 호환 CMP 연동 권장

## 8. KPI (광고 수익 관점 포함)

- 트래픽/참여
  - 일일 UV, PV, 세션 당 PV
  - 평균 체류시간(목표 120s+)
  - SNS 공유율(목표 30~40%)
- 수익/최적화
  - Monetizable Pageviews (MPV)
  - EPMV (USD) — 월별 추적, A/B로 개선 목표 설정
  - CTR, RPM(eRPM), viewability
  - AdSense policy violation count = 0

## 9. 예시 수익 시나리오 (가정 기반, 참고용)

- 가정: 일일 UV 10,000 / 세션당 PV 2 → 일일 PV 20,000
- 가정: 광고가 게재되는 페이지뷰 75% → MPV = 15,000
- 가정: 평균 eCPM(보수적) = $2.0
- 계산: 일일 광고수익 = (15,000 / 1000) \* $2 = $30 → 월 approx = $900
  - 메모: eCPM은 지역, 시즌, 광고유형에 따라 크게 변동. 초기에는 AdSense + 헤더비딩으로 eCPM 개선 필요.

## 10. 도입 리스크 및 완화책

- 정책 리스크: AdSense 정책 위반 시 계정 정지 — 완화: 광고 배치 가이드라인 문서화 및 사전 체크 프로세스
- 트래픽 리스크: 초기 유입 미달 — 완화: 인플루언서/UGC 캠페인 집중 투자, SEO 자동화
- 속도 리스크: 광고로 인한 LCP/CLS 저하 — 완화: lazy-load, 글꼴 최적화, ad refresh 제한

## 11. 실행 로드맵(런칭 0~3개월)
```
